# Bienvenido

Esta carpeta contiene la documentación oficial del proyecto **Academia de Gloria**.

Su objetivo es mantener organizada toda la información relacionada con la arquitectura, desarrollo, planificación, estándares, funcionalidades y evolución del proyecto.

La documentación forma parte integral de la Academia y evoluciona conjuntamente con el código fuente.

---

# Objetivos de la Documentación

- Centralizar el conocimiento del proyecto.
- Documentar las decisiones de arquitectura.
- Mantener la trazabilidad de los cambios.
- Facilitar el mantenimiento y evolución.
- Garantizar consistencia entre desarrollo y documentación.
- Servir como referencia técnica y funcional.

---

# Principio de Organización

La documentación de la Academia sigue el principio de **Responsabilidad Única**.

Cada documento responde a una única pregunta y evita duplicar información existente en otros documentos.

| Documento | Pregunta que responde |
|-----------|-----------------------|
| README | ¿Dónde está la documentación? |
| MASTER_PLAN | ¿Qué es la Academia y hacia dónde va? |
| PROJECT_MAP | ¿Cómo está organizado el proyecto? |
| ROADMAP | ¿Qué construiremos y cuándo? |
| CHANGELOG | ¿Qué ha cambiado? |
| DECISION_LOG | ¿Por qué se tomaron determinadas decisiones? |

Cuando un tema requiera mayor nivel de detalle, deberá desarrollarse en su documento específico, manteniendo este README únicamente como punto de entrada a la documentación oficial.

---


| Documento         | Propósito                                   |
| ----------------- | ------------------------------------------- |
| `README.md`       | Índice de la documentación.                 |
| `MASTER_PLAN.md`  | Visión y estrategia del proyecto.           |
| `ROADMAP.md`      | Evolución funcional por versiones.          |

| `CHANGELOG.md`    | Qué cambió en cada versión.                 |
| `DECISION_LOG.md` | Por qué se tomaron determinadas decisiones. |

# Convenciones

Como regla general, toda nueva funcionalidad importante deberá incluir:

- Código fuente.
- Documentación funcional.
- Registro en CHANGELOG.md.
- Actualización del MASTER_PLAN.md cuando corresponda.
- Actualización del PROJECT_MAP.md si cambia la estructura del proyecto.
- Actualización del ROADMAP.md cuando afecte la planificación.
